# 後端架構設計說明

本文件旨在闡述 `GFmotor` 後端應用程式的整體架構設計、核心組件及運作流程。

---

## 1. 語言與框架 (Language & Framework)

- **程式語言**: **Python 3.10+**
- **核心框架**: **FastAPI**
  - **原因**: FastAPI 是一個基於 Python 類型提示的現代高效能 Web 框架。其優勢包括：
    - **卓越性能**: 速度與 NodeJS 和 Go 並駕齊驅。
    - **快速開發**: 簡潔的語法能大幅提升開發效率。
    - **自動化文件**: 能根據程式碼自動生成互動式 API 文件 (Swagger UI 和 ReDoc)，完美符合 `API.md` 的文件化需求。
    - **資料驗證**: 內建 Pydantic，能進行強大的請求與回應資料驗證。

- **非同步伺服器 (ASGI)**: **Uvicorn**
  - **原因**: Uvicorn 是運行 FastAPI 應用的標準 ASGI 伺服器，能完全發揮其非同步性能。

---

## 2. 目錄結構 (Directory Structure)

為了確保程式碼的模組化與可維護性，後端將採用以下目錄結構：

```
/
|-- .env              # 用於儲存資料庫連線資訊、API 金鑰等敏感資料
|-- .env.sample       # .env 的設定範本
|-- requirements.txt  # Python 依賴套件列表
|-- main.py           # 🚀 應用程式主進入點
|
|-- api/              # API 路由層 (Endpoints)
|   |-- router.py       # 彙總所有路由
|   |-- endpoints/
|   |   |-- auth.py       # 登入、Google OAuth 驗證
|   |   |-- users.py      # 使用者資料管理
|   |   |-- products.py   # 商品管理
|   |   |-- bookings.py   # 預約管理
|   |   |-- orders.py     # 線上訂單管理
|   |   |-- admin.py      # 後台管理功能
|
|-- core/             # 核心組件
|   |-- config.py       # 讀取 .env 的環境變數
|   |-- security.py     # 密碼雜湊、JWT 權杖生成與驗證
|
|-- db/               # 資料庫存取層
|   |-- database.py     # 資料庫連線與 Session 管理
|   |-- models.py       # SQLAlchemy ORM 模型 (對應資料庫表格)
|   |-- crud.py         # 封裝資料庫的 CRUD (增刪改查) 操作
|
`-- schemas/          # 資料驗證層 (Pydantic Schemas)
    |-- user.py         # 使用者相關的請求/回應模型
    |-- product.py      # 商品相關模型
    |-- order.py        # 訂單相關模型
    |-- token.py        # JWT 權杖模型
```

## 3. 核心組件說明

- **`main.py`**
  - **功能**: 整個應用的啟動入口。它會初始化 FastAPI 應用，掛載 `api/router.py` 中的主路由，並可配置 CORS (跨域資源共享) 等中介軟體 (Middleware)。

- **`api/` (路由層)**
  - **功能**: 處理所有 HTTP 請求的進入點。每個 `endpoints` 下的檔案都代表一組相關的功能。
  - **範例**: 當前端請求 `GET /products/1` 時，請求會被導向到 `api/endpoints/products.py` 中對應的函式來處理。

- **`core/` (核心組件)**
  - `config.py`: 負責讀取 `.env` 檔案中的設定，讓應用程式可以取得資料庫密碼、金鑰等資訊，同時避免將敏感資訊寫死在程式碼中。
  - `security.py`: 處理所有與安全相關的邏輯。
    - **密碼處理**: 使用 `passlib` 對管理員密碼進行加密 (雜湊) 和驗證。
    - **JWT 權杖**: 使用 `python-jose` 來生成 (Login 後) 與驗證 (後續請求) JSON Web Tokens。

- **`db/` (資料庫層)**
  - `database.py`: 使用 SQLAlchemy 建立與 MySQL 的連線，並管理資料庫會話 (Session)，確保與資料庫的通訊穩定可靠。
  - `models.py`: 將 `gfmotor.sql` 中的每一個資料表，用 Python 類別 (Class) 的形式來表示。這使得操作資料庫可以像操作物件一樣直觀，無需頻繁撰寫原生 SQL。
  - `crud.py`: 將所有資料庫操作（如：`取得所有商品`、`依據ID查找使用者`、`建立一筆新訂單`）封裝成一個個函式。這讓 API 路由層的邏輯更乾淨，專注於處理請求，而非資料庫細節。

- **`schemas/` (資料驗證層)**
  - **功能**: 使用 Pydantic 模型來定義 API 的請求和回應格式。
  - **範例**: `ProductCreate` 模型可以定義「建立商品」時，請求 Body 必須包含 `name` (字串) 和 `price` (整數)。如果前端傳來的資料格式不符，FastAPI 會自動回傳一個清晰的錯誤訊息。

## 4. 認證機制 (Authentication)

系統將提供兩種登入方式：

1.  **管理員 (`admins`)**:
    - **流程**:
      1.  管理員在後台登入頁面輸入帳號密碼。
      2.  前端將帳密發送到 `/login` API。
      3.  後端使用 `core/security.py` 中的函式驗證密碼是否正確。
      4.  驗證成功後，生成一個有時效性的 JWT 權杖並回傳給前端。
      5.  前端在後續請求需權限的 API 時，需在 HTTP Header 的 `Authorization` 欄位中帶上此權杖。

2.  **一般使用者 (`users`)**:
    - **流程**: 採用業界標準的 **Google OAuth 2.0**。
      1.  前端引導使用者點擊「使用 Google 登入」。
      2.  前端處理 Google 登入流程，並從 Google 獲得一個 `ID Token`。
      3.  前端將此 `ID Token` 發送到後端的 `/auth/google` API。
      4.  後端收到後，會向 Google 的伺服器驗證此 `ID Token` 的真偽。
      5.  驗證成功後，從 Token 中解析出使用者的 Google ID, Email, Name 等資訊。
      6.  後端在 `users` 資料表中查找該 Google ID。如果不存在，則自動為其註冊一個新帳號。
      7.  最後，後端生成屬於**我們自己系統**的 JWT 權杖，並回傳給前端，完成登入。

## 5. 依賴套件 (Dependencies)

以下是本架構需要安裝的主要 Python 套件，將會被記錄在 `requirements.txt` 中：

- `fastapi`: 核心 Web 框架。
- `uvicorn[standard]`: ASGI 伺服器。
- `sqlalchemy`: ORM，用於操作資料庫。
- `pymysql` 或 `mysqlclient`: 連接 MySQL 的驅動程式。
- `pydantic`: 用於資料驗證及 `core/config.py` 中的設定管理。
- `python-dotenv`: 用於讀取 `.env` 檔案。
- `passlib[bcrypt]`: 用於密碼雜湊。
- `python-jose[cryptography]`: 用於 JWT 權杖操作。
- `google-auth`: 用於驗證 Google ID Token。
