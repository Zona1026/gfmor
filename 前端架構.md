# 前端架構設計說明

本文件旨在闡述 `GFmotor` 前端應用程式的整體架構設計、技術選型及運作流程。

---

## 1. 框架與工具 (Framework & Tools)

- **核心框架**: **Vue 3**
  - **原因**: 採用最新的 Vue 3 版本，搭配 **Composition API** (組合式 API) 進行開發。這使得程式碼邏輯的組織、複用和可維護性都遠勝於舊的 Options API。

- **建構工具**: **Vite**
  - **原因**: Vite 是一個極其快速的現代化前端建構工具。它提供秒級的冷啟動、近乎即時的熱模組更新 (HMR)，能大幅提升開發體驗與效率。

- **路由管理**: **Vue Router**
  - **原因**: Vue 官方的路由管理器，用於處理應用程式的頁面導航 (SPA, Single-Page Application)。

- **狀態管理**: **Pinia**
  - **原因**: Vue 官方推薦的新一代狀態管理器，它比 Vuex 更輕量、更直觀，且與 Vue 3 的 Composition API 完美整合，擁有完整的 TypeScript 支援。

- **HTTP 客戶端**: **Axios**
  - **原因**: 一個成熟穩定、功能強大的 Promise-based HTTP 客戶端，用於與後端 API 進行通訊。將透過攔截器 (interceptors) 實現請求的統一處理。

- **CSS 預處理器**: **SASS / SCSS**
  - **原因**: 提供變數、嵌套、混合 (Mixin) 等高級 CSS 功能，非常適合用於管理專案的主題色（灰紅色調）和複雜樣式。

## 2. 目錄結構 (Directory Structure)

為了確保專案的模組化與可擴展性，前端將採用以下目錄結構：

```
/
|-- .env              # 用於儲存環境變數 (如後端 API 的網址)
|-- index.html        # 應用程式的 HTML 進入點
|-- package.json      # 專案依賴與腳本設定
|-- vite.config.js    # Vite 的設定檔
|
|-- public/           # 靜態資源 (不會被建構流程處理的檔案，如 logo)
`-- src/              # 核心原始碼目錄
    |-- App.vue         # 根組件
    |-- main.js         # 🚀 應用程式主進入點
    |
    |-- api/            # API 請求層
    |   |-- index.js      # Axios 實例化與攔截器設定
    |   |-- auth.js       # 驗證相關的 API 請求
    |   |-- products.js   # 商品相關的 API 請求
    |
    |-- assets/         # 靜態資源 (會被建構流程處理的檔案)
    |   |-- main.scss     # 主要樣式表
    |   |-- _variables.scss # SASS 變數 (定義顏色、字體等)
    |
    |-- components/     # 可複用的 UI 組件
    |   |-- common/       # 通用基礎組件 (如 Button, Modal)
    |   |-- layout/       # 佈局組件 (如 Navbar, Footer)
    |   |-- ProductCard.vue # 業務相關組件範例
    |
    |-- router/         # 路由層
    |   |-- index.js      # 定義所有頁面路由與導航守衛
    |
    |-- store/          # 全域狀態管理層 (Pinia)
    |   |-- auth.js       # 驗證狀態 (使用者資訊、權杖)
    |   |-- cart.js       # 購物車狀態
    |
    `-- views/          # 頁面級組件
        |-- Home.vue
        |-- Login.vue
        |-- Products.vue
        |-- Admin/
        |   |-- Dashboard.vue
```

## 3. 核心組件說明

- **`main.js`**
  - **功能**: 初始化 Vue 應用，並掛載 `App.vue` 根組件。同時，它會安裝並啟用 `Vue Router` 和 `Pinia` 等核心插件。

- **`/api` (API 請求層)**
  - **功能**: 統一管理所有對後端 API 的請求。
  - **`index.js`**: 在此建立一個全域的 Axios 實例，並設定 `baseURL` (從 `.env` 讀取)。最重要的是，會在此設定**請求攔截器 (Request Interceptor)**，讓每次發送請求時，自動從 Pinia store 讀取 JWT 並加入到 HTTP Header 中。同時也會設定**回應攔截器 (Response Interceptor)** 來統一處理如 401 未授權等錯誤。

- **`/assets` (樣式資源)**
  - **功能**: 存放全域樣式檔案。
  - **`_variables.scss`**: 此檔案是實現主題色的關鍵。專案所需的**灰、紅色調**將會在此定義成 SASS 變數。所有組件都將引用這些變數而非寫死的色碼，以確保風格統一且易於修改。

- **`/components` (可複用組件)**
  - **功能**: 存放可以在不同頁面中重複使用的 UI 組件，如下拉式選單、卡片、導覽列等，提高開發效率。

- **`/router` (路由層)**
  - **功能**: 定義應用中所有頁面的路徑。例如，`/` 對應 `Home.vue`，`/products` 對應 `Products.vue`。
  - **導航守衛 (Navigation Guards)**: 會在此設定路由守衛，用於權限控制。例如，在進入 `/admin` 路徑前，守衛會檢查使用者是否已登入且具備管理員權限，否則將其重導向至登入頁面。

- **`/store` (全域狀態管理)**
  - **功能**: 使用 Pinia 集中管理整個應用的共享狀態。
  - **`auth.js`**: 儲存使用者的登入狀態、個人資訊以及後端發放的 JWT 權杖。為了讓使用者在重新整理頁面後仍保持登入，JWT 權杖會被同步儲存到瀏覽器的 `localStorage` 中。
  - **`cart.js`**: 管理購物車中的商品列表、數量等狀態。

- **`/views` (頁面組件)**
  - **功能**: 代表應用程式中的主要頁面，每個檔案都是一個完整的頁面。它們通常由多個 `/components` 中的小組件組合而成。

## 4. 認證流程 (Authentication Flow)

- **使用者登入 (Google Sign-In)**:
  1.  使用者點擊「使用 Google 登入」按鈕。
  2.  前端呼叫 Google 登入的相關套件 (如 `vue3-google-oauth2`)，彈出 Google 授權視窗。
  3.  使用者授權後，前端從 Google 獲取 `ID Token`。
  4.  前端透過 `src/api/auth.js` 中的函式，將此 `ID Token` 發送至後端進行驗證。
  5.  後端驗證成功後，回傳我們系統自己的 JWT 權杖。
  6.  前端接收到 JWT，將其與使用者資訊存入 Pinia (`store/auth.js`)，並同步將 JWT 寫入 `localStorage`。
  7.  登入完成。之後所有需要權限的 API 請求，都會由 Axios 攔截器自動附上 JWT。

- **使用者登出**:
  1.  呼叫 `store/auth.js` 中的登出動作 (action)。
  2.  清除 Pinia 中的使用者狀態與 JWT。
  3.  從 `localStorage` 中移除 JWT。
  4.  將使用者導向至首頁或登入頁。
