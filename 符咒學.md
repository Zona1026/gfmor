【專案背景與目標】 這是一個專為「機車改裝車行」設計的進銷存與維修管理系統。請根據我提供的開發文件進行系統開發。

【開發環境與架構】

後端： Python ，需讀取 `.env` 配置。

前端： Vue.js，UI 視覺請以「深灰色與熱血紅」為主色調，營造工業運動感。

資料庫： MySQL。

開發架構： 請嚴格遵循我提供的 `前端架構.md` 與 `後端架構.md` 進行開發。

【程式碼撰寫規範】

極致註解： 程式碼需包含大量且淺顯易懂的繁體中文註解。請假設讀者「沒有程式基礎」，解釋該段代碼的邏輯（例如：為什麼要這樣處理、這行在資料庫存了什麼）。

安全性： 嚴禁將 API Key、資料庫密碼等敏感資訊寫死在程式中。請提供 `.env.sample` 範本。

【文件維護任務】 每當開發完成一個階段，請同步更新或生成以下文件：

`使用說明.md`： 詳細教學如何安裝依賴（pip/npm）、如何啟動後端伺服器與前端專案。

`API.md`： 紀錄所有 API 介面（含網址、功能、Request/Response JSON 範例）。

`資料庫.md`： 包含 Table 名稱、功能、欄位細節（型態、索引、關聯）。

問題管理： 我會將遇到的問題紀錄於 `問題清單.md`；若問題解決，請將其移至 `已解決清單.md`。

【溝通偏好】 請全程使用繁體中文對話。
